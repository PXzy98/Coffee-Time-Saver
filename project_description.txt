This project was developed under the supervision of Shared Services Canada, which presented four user stories as potential directions for an AI integration prototype in large government organizations. The four options were: Option 1, The Daily Project Manager, which asks for a daily assistant that consolidates meetings, emails, chats, and task lists into a single morning view; Option 2, The Project Risk Checker, which asks for a risk simulator that reads project documentation and meeting transcripts to surface inconsistencies between what is written and what is actually said; Option 3, The Scope Creep Prevention, which asks for an agent that monitors communications and alerts when requests deviate from the original statement of work; and Option 4, The Legacy Data Management, which asks for an agent that transforms legacy documentation across formats into an interactive dependency map. Our team chose to implement Options 1 and 2.

The challenge Shared Services Canada posed is that government project managers cannot rely on clean, real-time, unified data. Unstructured information is locked in Word documents, emails, PDFs, and meeting transcripts, spread across siloed systems with strict access controls. Any AI assistant built for this environment must work with raw, fragmented inputs, must be auditable, and must not require connection to live departmental systems. Coffee Time Saver was built to meet exactly these constraints.

The system is a bilingual (EN/FR) web application that ingests raw documents (PDFs, Word files, spreadsheets, plain text) alongside emails and meeting transcripts, and consolidates everything into a single prioritized view every morning. A natural-language briefing, written in genuine AI-generated prose rather than a template, tells the project manager what happened, what requires follow-up, and what to prepare next. Tasks extracted from documents and emails are automatically ranked by the LLM based on actual importance, not just a numeric field. When a new file finishes processing, the result appears in the browser in real time via WebSocket, with no page refresh required.

The second capability targets a harder problem: risk that does not appear in formal status reports. The risk analyzer reads every document and email in a project, runs a layered LLM pipeline to summarize and cross-reference them, and then compares document pairs to surface contradictions between what is written in project plans and what is described in meeting transcripts or emails. Every identified risk includes the direct quotes from source material that support the finding, along with a confidence score adjusted by evidence density. The full report can be exported as a PDF or DOCX for steering committee distribution.

The design prioritizes reliability in constrained environments. All LLM calls, file parsing, and email polling run asynchronously through Celery workers so that the API never blocks. The LLM layer is provider-agnostic, working with OpenAI-compatible APIs, Claude, or a local Ollama instance, and every feature degrades gracefully to a rule-based fallback if no LLM is configured. Uploaded files are deleted after text extraction to avoid document retention concerns, and all write operations are logged to an audit table. The result is a system that brings genuine AI assistance to document-heavy, compliance-sensitive environments without requiring clean data or new vendor integrations.
